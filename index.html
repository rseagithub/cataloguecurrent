RSEA Catalogue Workflow Visualisations

Animated walkthroughs of the current and future-state catalogue management processes. Each animation steps through the screens an operator interacts with, the system hand-offs, and the resulting end-user view.
As-Is · Current State

Catalogue Creation Workflow
via Commerce Vision

Commerce Vision → Pronto Xi → Customer

Operator builds the catalogue and category hierarchy in the Commerce Vision CMS, exports the data, then enters product-line records into Pronto Xi (Inventory → Web CSS Menu → Web Catalogue). An overnight sync pushes the data back to Commerce Vision, where end users see it in the storefront.

To-Be · Future State

Catalogue Creation Workflow
via Adobe Commerce

Pronto Xi → Adobe Commerce → Customer

Pronto Xi becomes the master system. Operator creates the catalogue, defines a 3-level category hierarchy and assigns product lines (including style-based entries that explode into multiple SKUs) via new custom screens under Inventory → Adobe Catalogue Management. Includes a brief overview of Entitlements and Uniforms.
